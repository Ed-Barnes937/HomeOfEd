# Handoff: HomeOfEd — homepage v2 ("gallery" direction)

## Overview
A refresh of the **HomeOfEd** landing page (`homeofed.com`, the `hub` app). Same
job as today — wordmark on one side, the list of apps on the other — but the app
list becomes a **gallery of cards**, each with a small **live animated preview**
of what the app does. The palette shifts to a warmer paper tone and the type
changes to **Space Grotesk**.

Layout at a glance:
- **Left / main area:** wordmark "home *of* ed", a one-line lede, and a small
  "Made with <3" footer, all anchored bottom-left.
- **Right rail:** a fixed 540px column, divider on its left edge, holding an
  eyebrow (`THINGS TO PLAY WITH · N`) above a two-column grid of app cards.
- **Theme toggle:** top-**left** moon/sun button (it was top-right before).

It keeps light/dark theming (system preference on first load, manual toggle,
persisted to `localStorage`).

## Carried over from the current site — do NOT adopt the mock's versions

Two things stay exactly as they are in the live `apps/hub` implementation. The
mock re-implements both in a simpler/different way; **ignore the mock there** and
port the existing code across.

1. **The "of" hop animation.** Keep the runtime-measured hop from
   [`apps/hub/src/pages/HomePage.tsx`](../../apps/hub/src/pages/HomePage.tsx)
   (`useHopAnimation`) + the `.wordmark/.letter/.gap/.of` rules in
   [`HomePage.module.scss`](../../apps/hub/src/pages/HomePage.module.scss). The
   "of" enters from above and hops across the tops of **h → o → m → e** before
   parking in the gap; keyframes are generated by measuring each letter's centre,
   it re-runs on resize / after fonts load, and it respects
   `prefers-reduced-motion`.
   - The mock instead uses a trivial 1s `ofPop` bounce on an inline `of` span
     (`.of-anim` in the bundle). Discard that.
   - **Consequence for markup:** the wordmark must keep its per-letter structure
     (`h`, `o`, `m`, `e` spans + a `.gap` spacer + the absolutely-positioned
     `.of`) so the measured animation still works — the mock's flat
     `home<span>of</span>ed` markup will not drive the hop.

2. **The handmade-paper textured background.** Keep the real `/paper.webp`
   texture from `HomePage.module.scss` — the `.home::before` (grayscale +
   `soft-light`, `--paper-grain`) and `.home::after` (saturated + `multiply`,
   `screen` in dark, `--paper-fleck`) layers over the tinted `--bg`.
   - The mock fakes texture with three `radial-gradient` flecks plus an inline
     `feTurbulence` SVG noise tile (see the `#page` `background-image` in the
     template). Discard that; use the photo texture we already ship.

Everything else on the page should be updated to match the mock.

## About the design file
`home-of-ed.html` is a **self-unpacking bundle**, not plain source: the visible
markup is stored as JSON inside `<script type="__bundler/template">` and the
fonts/assets as a base64 (gzipped) manifest that a bootstrap script inflates via
`DecompressionStream` and swaps into the document on load. So:
- **To view it:** open in a modern browser with JS enabled (it needs
  `DecompressionStream`). It renders the warm paper page with the live canvas
  previews animating.
- **To read the real HTML/CSS:** extract the `__bundler/template` script's JSON
  and unescape it — that string is the actual page (fonts, inline styles, the
  `<x-dc>` markup, and a trailing `text/x-dc` component script with the app list
  and the canvas preview logic).

It's a **design reference**, not code to ship. Recreate it in `apps/hub` (React +
SCSS modules, as today), reusing our own primitives and the two carried-over
pieces above.

## Fidelity
**High-fidelity** for layout, palette, typography, card treatment, and the
preview animations — match them. The two carry-overs (hop animation, paper
texture) override the mock where they conflict.

#### Components

**App card (`.gcard`)**

**Status badge**

  then `LIVE` — 600 / 10px / `letter-spacing:.1em`, colour `--live`.

**Theme toggle (`.tgl`)**
- 34px transparent circle, `1px solid var(--toggle-border)`.
- **Moon** (light mode): 16px circle drawn with `box-shadow: inset -6px -3px 0 0

  `--toggle-fill` circle. The mock swaps them via `[data-theme='dark']`.

**Canvas previews** (one per `kind`, drawn on a `<canvas>` with DPR scaling; the

- `idle` — a pulsing dashed ring over `IN THE WORKS` (used for SOON apps).

All previews recolour from the theme (`accent()` / `pal()` in the mock).

## Interactions & Behavior
- **Theme:** first load reads `localStorage['homeofed-theme']`, falling back to
  `prefers-color-scheme`; the toggle flips it and **persists** the choice.
  `background-color` / `color` and card colours transition `.45s`.
- **Card hover:** lift + shadow (`.18s`). No hover state on SOON cards beyond the
  shared lift.
- **Previews:** start shortly after mount (mock delays ~90ms) and loop via
  `requestAnimationFrame` / interval.
- **"of" hop:** as carried over — fire-once on mount, reduced-motion aware.

## State
- `theme: 'light' | 'dark'` — from `localStorage` → system preference, toggled.
- App list is **static** content (see table). No data fetching for the UI. (`hub`
  still keeps its `trpc.health()` round-trip server-side as the worked DB example;
  it isn't surfaced on the page.)
